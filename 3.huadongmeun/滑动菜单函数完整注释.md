## 1. 先理解这个动画的本质

这个滑动菜单的核心思想不是“框在移动”，而是：

- 选中框固定在屏幕中间
- 左右一排图标整体移动
- 动画结束后，新的图标停到中间

所以代码里要同时处理两件事：

- 菜单逻辑：当前到底选中了哪一项
- 动画显示：图标这一帧应该画在什么位置

## 2. 原始代码加完整注释版

```c
// 滑动菜单函数
// 图标移动，上次选择的选项变为这次选择的选项，框本身不发生变化

uint8_t pre_selection;      // 上一次处于中间位置的图标索引
uint8_t target_selection;   // 这次动画想要移动到的目标图标索引
uint8_t x_pre = 48;         // 当前中间图标的 x 坐标，48 表示位于中间
uint8_t Speed = 4;          // 每次刷新移动的像素数，越大越快，但过大会不流畅
uint8_t move_flag;          // 动画开始标志位，0 表示不移动，1 表示执行移动动画


// 外部循环不断调用本函数
// 每调用一次，就重新清屏、重画一帧，因此形成动画效果
void Menu_Animation(void)
{
    OLED_Clear();                               // 先清屏，准备重画整帧画面
    OLED_ShowImage(42,10,44,44,Frame);         // 画出固定在中间的选中框

    // ---------------------------
    // 第一部分：处理动画位移逻辑
    // ---------------------------

    // 如果目标索引比当前索引大
    // 说明目标图标在右边
    // 为了让右边图标进入中间，整排图标需要向左移动
    if(pre_selection < target_selection)
    {
        x_pre -= Speed;                        // 当前中间图标每次向左移动 Speed 个像素

        // 当 x_pre 从 48 移动到 0
        // 说明原来中间的图标已经滑到左边一格的位置
        // 右边那个目标图标此时正好来到中间
        if(x_pre == 0)
        {
            pre_selection++;                   // 更新中间图标索引：中间图标变成右边那个
            move_flag = 0;                     // 动画结束，停止移动
            x_pre = 48;                        // x 坐标恢复到中间标准位置，准备下次动画
        }
    }

    // 如果当前索引比目标索引大
    // 说明目标图标在左边
    // 为了让左边图标进入中间，整排图标需要向右移动
    if(pre_selection > target_selection)
    {
        x_pre += Speed;                        // 当前中间图标每次向右移动 Speed 个像素

        // 当 x_pre 从 48 移动到 96
        // 说明原来中间的图标已经滑到右边一格的位置
        // 左边那个目标图标此时正好来到中间
        if(x_pre == 96)
        {
            pre_selection--;                   // 更新中间图标索引：中间图标变成左边那个
            move_flag = 0;                     // 动画结束
            x_pre = 48;                        // 恢复到中间标准位置
        }
    }


    // ---------------------------
    // 第二部分：根据当前中间图标，绘制左右相邻图标
    // ---------------------------
    // 整个界面始终以 pre_selection 为中心进行显示
    // 中间是 pre_selection
    // 左边依次是 pre_selection - 1、pre_selection - 2
    // 右边依次是 pre_selection + 1、pre_selection + 2

    // 如果当前中间图标左边至少还有 1 个图标，就把它画出来
    // x_pre - 48 表示它在中间图标左边一格
    if(pre_selection >= 1)
    {
        OLED_ShowImage(x_pre - 48, 16, 32, 32, Menu_Graph[pre_selection - 1]);
    }

    // 如果当前中间图标左边至少还有 2 个图标，就再画更左边那个
    // x_pre - 96 表示在中间图标左边两格
    if(pre_selection >= 2)
    {
        OLED_ShowImage(x_pre - 96, 16, 32, 32, Menu_Graph[pre_selection - 2]);
    }

    // 画当前处于中间位置的图标
    OLED_ShowImage(x_pre, 16, 32, 32, Menu_Graph[pre_selection]);

    // 下面两句是画右边的图标
    // 注意：如果菜单数量有限，这里最好也加边界判断
    // 否则 pre_selection 靠近最后一项时可能会数组越界
    OLED_ShowImage(x_pre + 48, 16, 32, 32, Menu_Graph[pre_selection + 1]);
    OLED_ShowImage(x_pre + 96, 16, 32, 32, Menu_Graph[pre_selection + 2]);

    OLED_Update();                             // 把缓冲区内容真正刷新到 OLED 屏幕
}


// 按键设定函数
// 作用：把本次动画的“起始图标”和“目标图标”设置好，然后调用动画函数
void Set_Selection(uint8_t move_flag, uint8_t Pre_Selection, uint8_t Target_Selection)
{
    if(move_flag == 1)
    {
        pre_selection = Pre_Selection;         // 记录本次动画开始时中间是谁
        target_selection = Target_Selection;   // 记录本次动画要去哪里

        // 外部循环会不断调用这个函数
        // 所以 Menu_Animation() 会被反复刷新，从而形成连续动画
        Menu_Animation();
    }
}


uint8_t menu_flag = 1;                         // 当前选中的菜单编号，通常按 1~7 编号

int Menu(void)
{
    move_flag = 1;                             // 初始默认允许移动
    uint8_t DirectFlag = 2;                    // 1 表示上一项，2 表示下一项

    while(1)
    {
        // ---------------------------
        // 第一部分：按键扫描
        // ---------------------------
        KeyNum = Key_GetNum();                 // 读取按键值
        uint8_t menu_flag_temp = 0;            // 临时变量，用于确认进入哪个子菜单

        if(KeyNum == 1)
        {
            DirectFlag = 1;                    // 按键 1：切换到上一项
            move_flag = 1;                     // 允许播放动画
            menu_flag--;                       // 当前菜单编号减 1

            // 如果减到 0，说明越过最前面了，回到最后一项
            if(menu_flag <= 0) menu_flag = 7;
        }
        else if(KeyNum == 2)
        {
            DirectFlag = 2;                    // 按键 2：切换到下一项
            move_flag = 1;                     // 允许播放动画
            menu_flag++;                       // 当前菜单编号加 1

            // 如果超过最后一项，就回到第 1 项
            if(menu_flag >= 8) menu_flag = 1;
        }
        else if(KeyNum == 3)
        {
            // 确认进入当前菜单项
            OLED_Clear();
            OLED_Update();
            menu_flag_temp = menu_flag;
        }

        // ---------------------------
        // 第二部分：根据确认项进行页面跳转
        // ---------------------------
        if(menu_flag_temp == 1) { return 0; }
        else if(menu_flag_temp == 2) {}
        else if(menu_flag_temp == 3) {}
        else if(menu_flag_temp == 4) {}
        else if(menu_flag_temp == 5) {}
        else if(menu_flag_temp == 6) {}
        else if(menu_flag_temp == 7) {}


        // ---------------------------
        // 第三部分：根据方向决定动画起点和终点
        // ---------------------------
        if(menu_flag == 1)
        {
            // 这是特殊情况：当前逻辑菜单为第 1 项
            // DirectFlag == 1 表示从第 2 项滑到第 1 项
            // DirectFlag == 2 表示保持在最左边，不需要左边更多图标
            if(DirectFlag == 1) Set_Selection(move_flag, 1, 0);
            else if(DirectFlag == 2) Set_Selection(move_flag, 0, 0);
        }
        else
        {
            // 一般情况
            // 如果往上一项移动：动画从 menu_flag 对应的右边项滑到左边项
            if(DirectFlag == 1) Set_Selection(move_flag, menu_flag, menu_flag - 1);

            // 如果往下一项移动：
            // 由于 menu_flag 已经先自增过，所以这里要倒推回动画前的中心项
            else if(DirectFlag == 2) Set_Selection(move_flag, menu_flag - 2, menu_flag - 1);
        }
    }
}
```

## 3. 这段代码最难的点在哪里

难点不在 OLED，也不在 `while(1)`，而在于它混用了两套“编号体系”：

- `menu_flag` 用的是“第 1 项、第 2 项、第 3 项”
- `Menu_Graph[]` 用的是数组索引“0、1、2、3”

这就导致这里看起来很绕：

```c
else if(DirectFlag==2)Set_Selection(move_flag,menu_flag-2,menu_flag-1);
```

它不是乱写，而是在做“编号转换”：

- `menu_flag` 已经先 `++`
- 所以要减回来，得到动画开始时的图标索引

---

## 4. 用一个具体例子理解

假设当前选中第 3 个菜单，用户按“下一项”：

1. 原来 `menu_flag = 3`
2. 按右键后，`menu_flag++`，变成 `4`
3. 但动画实际上是“从第 3 项滑到第 4 项”
4. 如果转成数组索引，就是“从索引 2 滑到索引 3”
5. 所以代码写成：

```c
Set_Selection(move_flag, menu_flag - 2, menu_flag - 1);
```

也就是：

```c
Set_Selection(move_flag, 2, 3);
```

这时就对上了。

---

## 5. 左右图标显示是怎么排的

假设当前：

```c
pre_selection = 3;
x_pre = 48;
```

那么屏幕上这几个图标的位置大致是：

- `pre_selection - 2` 画在 `x = -48`
- `pre_selection - 1` 画在 `x = 0`
- `pre_selection` 画在 `x = 48`
- `pre_selection + 1` 画在 `x = 96`
- `pre_selection + 2` 画在 `x = 144`

也就是说，程序始终在画“中间图标 + 左右相邻图标”，然后通过修改 `x_pre` 让整排一起移动。

---

## 6. 右边为什么也应该加判断

当前原代码左边做了判断：

```c
if(pre_selection >= 1)
if(pre_selection >= 2)
```

是为了防止：

- `pre_selection - 1`
- `pre_selection - 2`

变成负数索引。

但右边原代码没有判断：

```c
OLED_ShowImage(x_pre + 48,16,32,32,Menu_Graph[pre_selection + 1]);
OLED_ShowImage(x_pre + 96,16,32,32,Menu_Graph[pre_selection + 2]);
```

如果菜单总数只有 7 项，索引范围是 `0~6`，那么：

- `pre_selection = 5` 时，`pre_selection + 2 = 7`，越界
- `pre_selection = 6` 时，`pre_selection + 1 = 7`，越界

可能出现：

- 图标乱码
- 花屏
- 读到错误数据
- 严重时程序异常

更安全的写法应该是：

```c
if(pre_selection + 1 < MENU_COUNT)
{
    OLED_ShowImage(x_pre + 48,16,32,32,Menu_Graph[pre_selection + 1]);
}

if(pre_selection + 2 < MENU_COUNT)
{
    OLED_ShowImage(x_pre + 96,16,32,32,Menu_Graph[pre_selection + 2]);
}
```

## 7. 整体一句话总结

这个滑动菜单系统可以概括成：

- `menu_flag` 决定逻辑上当前选中哪一项
- `pre_selection` 决定当前中间显示的是哪一个图标
- `target_selection` 决定动画要滑到哪一个图标
- `x_pre` 决定这一帧图标实际画在什么位置
- 框不动，动的是整排图标
