# 菜单逻辑图

这份文档对应项目里的菜单相关代码，主要帮助理解：

- `Menu()` 主循环在做什么
- `menu_flag`、`DirectFlag`、`pre_selection`、`target_selection` 分别表示什么
- 按键后菜单是怎么切换的
- 滑动动画是怎么被触发的

---

## 1. 一句话先理解

这套菜单逻辑本质上就是：

```text
按键输入
  -> 改变当前菜单编号
  -> 算出动画从哪张图滑到哪张图
  -> 调用动画函数刷新屏幕
  -> 新图标停在中间
```

---

## 2. 变量作用树状图

```text
menu_flag
├─ 含义：当前逻辑上选中了第几个菜单
├─ 范围：1 ~ 7
└─ 作用：决定当前最终应该停在哪个菜单

DirectFlag
├─ 含义：本次移动方向
├─ 1：上一项
└─ 2：下一项

pre_selection
├─ 含义：动画开始时，中间显示的图标索引
└─ 注意：它不是“中间位置坐标”，而是“中间位置上放的是哪张图”

target_selection
├─ 含义：动画结束后，中间要变成的图标索引
└─ 作用：决定动画目标

x_pre
├─ 含义：当前中间图标的横坐标
├─ x_pre = 48：标准中间位置
├─ x_pre = 0：当前图标滑到左边一格
└─ x_pre = 96：当前图标滑到右边一格

move_flag
├─ 含义：是否执行滑动动画
├─ 0：不移动
└─ 1：移动
```

---

## 3. 菜单编号和图标索引对应关系

```text
menu_flag = 1  -> 图标索引 0
menu_flag = 2  -> 图标索引 1
menu_flag = 3  -> 图标索引 2
menu_flag = 4  -> 图标索引 3
menu_flag = 5  -> 图标索引 4
menu_flag = 6  -> 图标索引 5
menu_flag = 7  -> 图标索引 6
```

注意：

- `menu_flag` 是“第几个菜单”
- `Menu_Graph[]` 用的是数组索引，从 `0` 开始

所以读代码时一定要注意这两套编号不要混了。

---

## 4. Menu() 主流程图

```text
进入 Menu()
│
├─ 初始化
│  ├─ move_flag = 1
│  └─ DirectFlag = 2
│
└─ while(1) 死循环
   │
   ├─ 读取按键：KeyNum = Key_GetNum()
   │
   ├─ 如果 KeyNum == 1
   │  ├─ 表示：选择上一项
   │  ├─ DirectFlag = 1
   │  ├─ move_flag = 1
   │  ├─ menu_flag--
   │  └─ 如果 menu_flag < 1
   │     └─ menu_flag = 7
   │
   ├─ 如果 KeyNum == 2
   │  ├─ 表示：选择下一项
   │  ├─ DirectFlag = 2
   │  ├─ move_flag = 1
   │  ├─ menu_flag++
   │  └─ 如果 menu_flag > 7
   │     └─ menu_flag = 1
   │
   ├─ 如果 KeyNum == 3
   │  ├─ 表示：确认当前菜单
   │  ├─ 清屏
   │  ├─ 刷新屏幕
   │  └─ 根据 menu_flag_temp 进入对应功能
   │
   ├─ 根据当前 menu_flag 和 DirectFlag
   │  └─ 调用 Set_Selection(...)
   │
   └─ Set_Selection(...)
      ├─ 设置 pre_selection
      ├─ 设置 target_selection
      └─ 调用 Menu_Animation()
```

---

## 5. 按键逻辑树状图

```text
按键输入
├─ 按键 1
│  ├─ 含义：上一项
│  ├─ menu_flag 减 1
│  └─ 如果减到 0
│     └─ 回到 7
│
├─ 按键 2
│  ├─ 含义：下一项
│  ├─ menu_flag 加 1
│  └─ 如果加到 8
│     └─ 回到 1
│
└─ 按键 3
   ├─ 含义：确认
   ├─ 记录当前 menu_flag
   └─ 进入对应页面
```

所以菜单是循环的：

```text
1 -> 2 -> 3 -> 4 -> 5 -> 6 -> 7 -> 1
1 <- 2 <- 3 <- 4 <- 5 <- 6 <- 7 <- 1
```

---

## 6. Set_Selection() 的作用

```text
Set_Selection(move_flag, A, B)
├─ A：动画开始时，中间显示的图标索引
├─ B：动画结束后，中间要变成的图标索引
└─ 作用：把“动画起点”和“动画终点”交给 Menu_Animation()
```

举例：

```text
Set_Selection(move_flag, 1, 0)
├─ 开始：中间是索引 1 的图标
└─ 结束：中间变成索引 0 的图标
```

这表示“从第 2 个图标滑到第 1 个图标”。

## 7. Menu_Animation() 动画流程图

```text
进入 Menu_Animation()
│
├─ OLED_Clear()
├─ 画固定边框 Frame
│
├─ 判断目标在左边还是右边
│  │
│  ├─ 如果 pre_selection < target_selection
│  │  ├─ 目标在右边
│  │  ├─ 整排图标向左移动
│  │  ├─ x_pre -= Speed
│  │  └─ 如果 x_pre == 0
│  │     ├─ pre_selection++
│  │     ├─ move_flag = 0
│  │     └─ x_pre = 48
│  │
│  └─ 如果 pre_selection > target_selection
│     ├─ 目标在左边
│     ├─ 整排图标向右移动
│     ├─ x_pre += Speed
│     └─ 如果 x_pre == 96
│        ├─ pre_selection--
│        ├─ move_flag = 0
│        └─ x_pre = 48
│
└─ 根据 pre_selection 画左右相邻图标
   ├─ 左2：pre_selection - 2
   ├─ 左1：pre_selection - 1
   ├─ 中间：pre_selection
   ├─ 右1：pre_selection + 1
   ├─ 右2：pre_selection + 2
   └─ OLED_Update()
```

## 8. 为什么看起来像“框没动，但菜单在动”

```text
固定框一直在中间
│
└─ 变化的是图标的位置
   ├─ 中间图标左右滑
   ├─ 左右相邻图标跟着一起滑
   └─ 所以看起来像整个菜单在左右平移
```

也就是说：

- 框不动
- 图标整体动

---

## 9. 图标排列示意图

假设当前：

```text
pre_selection = 3
x_pre = 48
```

那么显示关系大概是：

```text
屏幕上的位置：

x_pre-96    x_pre-48    x_pre      x_pre+48    x_pre+96
   │           │          │            │           │
   │           │          │            │           │
 index 1    index 2    index 3      index 4     index 5

也就是：
pre_selection-2
pre_selection-1
pre_selection
pre_selection+1
pre_selection+2
```

所以程序每次不是只画一个图标，而是画“中间 + 左右邻居”。

---

## 10. 你最容易混淆的点

```text
不要把下面两个东西搞混：

1. 中间的位置
   └─ 是固定的，靠 x_pre 控制显示坐标

2. 中间显示的是哪张图
   └─ 是 pre_selection / target_selection 控制的
```

也就是说：

- `x_pre` 决定“画在哪”
- `pre_selection` 决定“画谁”

## 11. 当前代码中的特殊边界情况

```text
menu_flag == 1
├─ 是最左边菜单
└─ 需要单独处理

menu_flag == 7
├─ 是最右边菜单
└─ 如果继续按上一项/下一项，要特别注意数组索引是否越界
```

特别是：

```text
Menu_Graph[0] ~ Menu_Graph[6]
```

如果代码里出现：

```text
Menu_Graph[7]
Menu_Graph[8]
```

就会越界。

---

## 12. 最后用一句最通俗的话总结

```text
Menu() 负责决定“要选哪个菜单”
Set_Selection() 负责决定“动画从哪张图滑到哪张图”
Menu_Animation() 负责把这个滑动过程画出来
```

所以完整关系是：

```text
按键
 -> 改 menu_flag
 -> 算 pre_selection / target_selection
 -> 图标整体滑动
 -> 新图标停在中间
```

推荐按这个顺序看代码：

```text
第一步：只看 Menu()
  └─ 搞清楚按键怎么改 menu_flag

第二步：只看 Set_Selection()
  └─ 搞清楚它只是“传动画参数”

第三步：只看 Menu_Animation()
  └─ 搞清楚它怎么根据参数去画动画
```

这样最容易看懂。
