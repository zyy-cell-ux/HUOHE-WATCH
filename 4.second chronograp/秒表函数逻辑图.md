## 1. 秒表功能一句话理解

```text
进入秒表页面
 -> 显示当前计时值
 -> 按键控制开始 / 暂停 / 清零 / 返回
 -> 循环刷新显示
```

---

## 2. 秒表最常见的变量结构

```text
second_flag / stopwatch_flag
├─ 含义：是否正在计时
├─ 0：暂停状态
└─ 1：运行状态

minute / second / ms
├─ 含义：秒表当前计时值
├─ minute：分钟
├─ second：秒
└─ ms：毫秒或百分秒

KeyNum
└─ 含义：当前按键值

display_refresh_flag
└─ 含义：是否需要刷新显示
```

---

## 3. 秒表页面主流程图

```text
进入秒表函数
│
├─ 初始化显示界面
│  ├─ 显示标题“秒表”
│  ├─ 显示当前计时值 00:00.xx
│  └─ 显示按键提示
│
└─ while(1) 循环
   │
   ├─ 读取按键 KeyNum = Key_GetNum()
   │
   ├─ 如果正在运行
   │  └─ 根据定时基准累加时间
   │
   ├─ 根据按键执行状态切换
   │
   ├─ 刷新 OLED 显示
   │
   └─ 继续下一轮循环
```

## 4. 最典型的按键逻辑树状图

下面是最常见、也最适合这个项目结构的秒表按键逻辑：

```text
按键输入
├─ Key1
│  ├─ 如果当前暂停
│  │  └─ 开始计时
│  └─ 如果当前运行
│     └─ 暂停计时
│
├─ Key2
│  ├─ 如果当前暂停
│  │  └─ 清零
│  └─ 如果当前运行
│     └─ 也可能被设计成“计次 / 暂不处理”
│
└─ Key3
   └─ 退出秒表页面，返回上一级菜单
```

## 6. 如果用 while(1) 写，逻辑通常是这样

```text
while(1)
├─ 读按键
├─ 判断当前是不是运行状态
│  ├─ 是
│  │  └─ 更新时间
│  └─ 否
│     └─ 保持当前时间不变
│
├─ 判断 Key1
│  └─ 切换 运行 / 暂停
│
├─ 判断 Key2
│  └─ 清零
│
├─ 判断 Key3
│  └─ return 返回菜单
│
└─ 刷新显示
```

## 7. 时间累加的树状图

假设秒表最小单位是百分秒或毫秒，时间更新逻辑一般是：

```text
最小单位 +1
├─ 如果未满
│  └─ 直接显示
└─ 如果满了
   ├─ 最小单位清零
   ├─ 秒 +1
   ├─ 如果秒满 60
   │  ├─ 秒清零
   │  └─ 分钟 +1
   └─ 继续显示
```

例如：

```text
ms++
├─ if(ms >= 100)
│  ├─ ms = 0
│  ├─ second++
│  └─ if(second >= 60)
│     ├─ second = 0
│     └─ minute++
```

## 8. OLED 显示逻辑图

```text
显示秒表页面
├─ 显示标题
│  └─ 例如：秒表
├─ 显示时间
│  └─ 例如：00:12.35
├─ 显示状态
│  ├─ RUN
│  └─ PAUSE
└─ 调用 OLED_Update()
```

所以秒表页面通常每轮循环都会做：

```text
清屏 / 局部清屏
 -> 重新显示最新时间
 -> OLED_Update()
```

## 9. 秒表函数更清晰的树状流程图

```text
秒表函数 Stopwatch()
│
├─ 进入页面
│  ├─ 初始化显示
│  └─ 默认状态：暂停
│
└─ while(1)
   │
   ├─ 读按键
   │
   ├─ Key1 ?
   │  ├─ 是
   │  │  ├─ 当前暂停 -> 变成运行
   │  │  └─ 当前运行 -> 变成暂停
   │  │
   │  └─ 否
   │
   ├─ Key2 ?
   │  ├─ 是
   │  │  └─ 清零计时值
   │  └─ 否
   │
   ├─ Key3 ?
   │  ├─ 是
   │  │  └─ return 返回菜单
   │  └─ 否
   │
   ├─ 当前是否运行？
   │  ├─ 是
   │  │  └─ 更新时间
   │  └─ 否
   │     └─ 时间保持不变
   │
   └─ 刷新 OLED
```

## 10. 如果把它放回你的项目菜单里，通常关系是这样

```text
主菜单 Menu()
├─ 选中“秒表”图标
```
